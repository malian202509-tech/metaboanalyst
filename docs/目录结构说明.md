# 代谢组学Python分析工作区目录结构说明

本工作区专为胎盘氧化脂质组学数据分析设计，严格遵循SOP（标准作业流程）按阶段推进，数据流向单向不可逆（从 `data/01_raw` 最终产出到 `results/`），符合最高级别的生物信息学规范。

## 目录树架构

```text
代谢组学python分析/
├── data/                               # 🔢 数据存放区（按数据流向分层）
│   ├── 01_raw/                         # [绝对只读] 质谱下机原始峰面积表、原始临床Metadata Excel
│   ├── 02_preprocessed/                # [阶段1产物] 过完80%规则、插补、PQN归一化、log转换的矩阵
│   ├── 03_batch_corrected/             # [阶段2产物] ComBat/QC-RLSC去除批次与漂移后的干净矩阵
│   └── 04_cohorts/                     # [阶段3产物] 切分好的主分析组(含并发症)与纯净亚组队列数据
│
├── scripts/                            # 💻 分析代码区（严格按 SOP 阶段编号执行）
│   ├── utils/                          # 通用工具函数 (如SMD计算、绘图基础函数)
│   ├── 00_baseline_table1.py           # 【打地基】处理 Metadata，生成 Table1 和协变量 SMD 检查
│   ├── 01_qc_and_preprocessing.py      # 【阶段1】QC聚集度检查、缺失值截断插补、数据归一化及转换
│   ├── 02_batch_effect_correction.py   # 【阶段2】PCA批次诊断、去批次核查、输出校正后矩阵
│   ├── 03_main_glm_analysis.py         # 【阶段4】跑 limma/GLM 获取主分析差异物 (输出 P, BH-q, FC)
│   ├── 04_sensitivity_analysis.py      # 【阶段3/4】利用纯净亚组和排离群点跑敏感性分析并对比
│   └── 05_biological_interpretation.py # 【阶段5】家族分类 Heatmap、酶活性代理比值计算与 LION 富集
│
├── results/                            # 📊 结果输出区（随时可被代码覆盖重写）
│   ├── tables/                         # 输出 Table1, Table2 差异总表, S1-S4 补充表
│   └── figures/                        #
│       ├── 01_QC_and_Batch_PCA/        # 校正前后 PCA, 进样顺序漂移图
│       ├── 02_Volcano_plots/           # 各组 Volcano 图
│       ├── 03_Heatmaps/                # 氧化脂质家族聚类热图
│       ├── 04_Metabolite_boxplots/     # 差异物质的点线箱型图及偏残差图 (Partial residual)
│       └── 05_Enrichment/              # 富集气泡图 (Bubble plots)
│
├── docs/                               # 📝 文档区
│   ├── 胎盘氧化脂质组学分析流程SOP_增强版.md  # SOP 归档
│   ├── 数据处理与分析进展.md                  # 开发与分析日志
│   └── 目录结构说明.md                        # 本文件的存放位置
```

## 核心设计思路：

1. **绝对隔离的原数据 (`data/01_raw/`)**：不管后面的代码怎么报错崩溃，原数据仅被读取。永远不会污染最原始的数据。
2. **流水线运行 (`scripts/`)**：每日工作可从具体编号的脚本入手，如 `00_baseline_table1.py` 负责输出基线表；此脚本处理后的中间文件传给下个阶段。每个进程解耦，免去每次洗数据的困扰。
3. **主辅双轨并行 (`data/04_cohorts/`)**：切分不同子队列，存放主队列以及剔除合并症干扰纯净队列的中间数据，为交叉验证与敏感性比较留足空间。
